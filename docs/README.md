# FluentDraft Docs

This folder contains the source-of-truth planning documents for FluentDraft.

## Reading Order

1. [plan.md](../plan.md) - MVP scope, product plan, assumptions, and test plan.
2. [system-design.md](./system-design.md) - Detailed end-to-end system behavior and key flows.
3. [architecture.md](./architecture.md) - High-level product and technical architecture.
4. [database.md](./database.md) - Supabase/Postgres data model direction.
5. [database-schema.md](./database-schema.md) - Proposed Supabase/Postgres tables, indexes, RLS direction, and ER diagram.
6. [api-contracts.md](./api-contracts.md) - Frontend, server, Supabase, and future-client contract shapes.
7. [tasks-and-acceptance-criteria.md](./tasks-and-acceptance-criteria.md) - Local task list with step order, dependencies, and definition of done.
8. [project-structure.md](./project-structure.md) - Planned Next.js code organization.
9. [style-guide.md](./style-guide.md) - Visual design direction, palette, UI rules, and anti-patterns.
10. [testing-strategy.md](./testing-strategy.md) - Unit, integration, E2E, manual QA, RLS, and release checks.

## External Tracker

The Notion `MVP Task Tracker` is the step-by-step execution tracker. Follow the `Step` order unless a task dependency requires otherwise.

## Maintenance Rule

When a product, architecture, data, testing, or design decision changes, update the relevant doc here. Do not leave important decisions only in chat or Notion comments.
