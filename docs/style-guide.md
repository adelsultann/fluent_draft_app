# FluentDraft Style Guide

## Design Direction

FluentDraft should feel like:

- Serious
- Practical
- Premium
- Calm
- Focused
- Professional
- Modern SaaS

It should not feel childish or like a typing game.

## Color Palette

- Primary navy: `#0F172A`
- Action blue: `#2563EB`
- Success green: `#22C55E`
- Phrase amber: `#F59E0B`
- Error red: `#EF4444`
- App background: `#F8FAFC`
- Card surface: `#FFFFFF`
- Main text: `#1E293B`
- Muted text: `#64748B`
- Border: `#E2E8F0`

## UI Rules

- Navy for structure, headers, and brand weight.
- Blue for primary actions and selected states.
- Green for success and completed practice.
- Red for mistakes and errors.
- Amber for useful phrases and learning highlights.
- White cards on off-white background as the default layout pattern.

## Component Guidance

- Primary buttons should use action blue.
- Destructive or mistake-related states should use error red.
- Correct answers, completion, and success feedback should use success green.
- Useful phrases and learning highlights should use phrase amber.
- Cards should feel clean and work-focused, not decorative.
- Progress indicators should support the learning loop without making the app feel like a speed game.
- Badges, levels, ranks, and missions should feel polished and professional.

## Practice Screen Guidance

The practice screen should feel like a focused writing workspace.

It should prioritize:

- Scenario context
- Goal
- Tone
- Current phase
- Useful phrases
- Typing and recall feedback
- Pronunciation feedback
- Completion and learning progress

Avoid making WPM visually dominant. FluentDraft is about better English writing, recall, pronunciation, and confidence.

## Anti-Patterns

Avoid:

- Childish game visuals.
- Typing-game styling.
- Overly playful illustrations.
- Noisy gradients.
- Decorative effects that distract from practice.
- UI dominated by WPM or speed.
- Cramped mobile layouts.
- Text overlap.
- Low-contrast feedback.
- Translation UI that distracts from English-first learning.
